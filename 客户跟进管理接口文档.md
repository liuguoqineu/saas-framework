## 1. 跟进记录管理

### 1.1 新增跟进记录

**接口描述**：录入跟进内容，支持关联客户、标注跟进状态

**请求方式**：POST

**请求路径**：`/api/v1/follow-up/records`

**请求头**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是 | Bearer Token认证 |

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customerId | Long | 是 | 关联客户ID |
| followUpTime | DateTime | 是 | 跟进时间，格式：yyyy-MM-dd HH:mm:ss |
| followUpPerson | String | 是 | 跟进人姓名 |
| followUpPersonId | Long | 是 | 跟进人ID |
| followUpMethod | Integer | 是 | 跟进方式：1-电话 2-微信 3-邮件 4-上门拜访 5-其他 |
| followUpContent | String | 是 | 跟进内容，最大500字 |
| nextPlan | String | 否 | 下一步计划，最大200字 |
| followUpStatus | Integer | 是 | 跟进状态：1-待跟进 2-已跟进 3-已达成意向 |
| attachments | Array | 否 | 附件列表 |
| attachments[].fileName | String | 否 | 附件名称 |
| attachments[].fileUrl | String | 否 | 附件URL |
| attachments[].fileSize | Long | 否 | 附件大小(字节) |

**请求示例**：
```json
{
  "customerId": 1001,
  "followUpTime": "2024-01-15 14:30:00",
  "followUpPerson": "张三",
  "followUpPersonId": 100,
  "followUpMethod": 1,
  "followUpContent": "与客户沟通了产品需求，客户对A产品比较感兴趣，预算在50万左右。",
  "nextPlan": "下周三上门演示产品",
  "followUpStatus": 2,
  "attachments": [
    {
      "fileName": "产品方案.pdf",
      "fileUrl": "https://xxx.com/files/xxx.pdf",
      "fileSize": 1024000
    }
  ]
}
```

**响应参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 响应码：200-成功 |
| message | String | 响应消息 |
| data | Object | 响应数据 |
| data.id | Long | 跟进记录ID |
| data.customerId | Long | 客户ID |
| data.customerName | String | 客户名称 |
| data.followUpTime | DateTime | 跟进时间 |
| data.followUpPerson | String | 跟进人 |
| data.followUpMethod | Integer | 跟进方式 |
| data.followUpMethodDesc | String | 跟进方式描述 |
| data.followUpContent | String | 跟进内容 |
| data.nextPlan | String | 下一步计划 |
| data.followUpStatus | Integer | 跟进状态 |
| data.followUpStatusDesc | String | 跟进状态描述 |
| data.createTime | DateTime | 创建时间 |

**响应示例**：
```json
{
  "code": 200,
  "message": "新增成功",
  "data": {
    "id": 2001,
    "customerId": 1001,
    "customerName": "某某科技有限公司",
    "followUpTime": "2024-01-15 14:30:00",
    "followUpPerson": "张三",
    "followUpMethod": 1,
    "followUpMethodDesc": "电话",
    "followUpContent": "与客户沟通了产品需求，客户对A产品比较感兴趣，预算在50万左右。",
    "nextPlan": "下周三上门演示产品",
    "followUpStatus": 2,
    "followUpStatusDesc": "已跟进",
    "createTime": "2024-01-15 14:35:00"
  }
}
```

### 1.2 查询跟进记录列表

**接口描述**：按客户、跟进人、跟进时间、跟进状态筛选查询跟进记录列表

**请求方式**：GET

**请求路径**：`/api/v1/follow-up/records`

**请求头**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是 | Bearer Token认证 |

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customerId | Long | 否 | 客户ID |
| customerName | String | 否 | 客户名称（模糊查询） |
| followUpPersonId | Long | 否 | 跟进人ID |
| followUpPerson | String | 否 | 跟进人姓名（模糊查询） |
| followUpStatus | Integer | 否 | 跟进状态：1-待跟进 2-已跟进 3-已达成意向 |
| followUpMethod | Integer | 否 | 跟进方式 |
| startTime | Date | 否 | 跟进时间开始，格式：yyyy-MM-dd |
| endTime | Date | 否 | 跟进时间结束，格式：yyyy-MM-dd |
| pageNum | Integer | 否 | 页码，默认1 |
| pageSize | Integer | 否 | 每页条数，默认10，最大100 |
| orderBy | String | 否 | 排序字段：followUpTime/createTime，默认followUpTime |
| orderType | String | 否 | 排序方式：asc/desc，默认desc |

**请求示例**：
```
GET /api/v1/follow-up/records?customerId=1001&followUpStatus=2&startTime=2024-01-01&endTime=2024-01-31&pageNum=1&pageSize=10
```

## 2. 跟进提醒管理

### 2.1 新增跟进提醒

**接口描述**：设置跟进提醒，支持短信/系统内提醒

**请求方式**：POST

**请求路径**：`/api/v1/follow-up/reminders`

**请求头**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是 | Bearer Token认证 |

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customerId | Long | 是 | 关联客户ID |
| followUpRecordId | Long | 否 | 关联跟进记录ID |
| reminderTime | DateTime | 是 | 提醒时间，格式：yyyy-MM-dd HH:mm:ss |
| reminderPersonId | Long | 是 | 提醒人ID |
| reminderPerson | String | 是 | 提醒人姓名 |
| reminderContent | String | 是 | 提醒内容，最大200字 |
| reminderType | Integer | 是 | 提醒方式：1-系统内提醒 2-短信提醒 3-系统内+短信 |
| reminderCycle | Integer | 否 | 提醒周期：0-单次 1-每天 2-每周 3-每月，默认0 |
| cycleEndTime | Date | 否 | 周期提醒结束时间 |

**请求示例**：
```json
{
  "customerId": 1001,
  "followUpRecordId": 2001,
  "reminderTime": "2024-01-20 09:00:00",
  "reminderPersonId": 100,
  "reminderPerson": "张三",
  "reminderContent": "请跟进某某科技客户，准备上门演示",
  "reminderType": 3,
  "reminderCycle": 0
}
```

**响应参数**：
| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 响应码：200-成功 |
| message | String | 响应消息 |
| data | Object | 响应数据 |
| data.id | Long | 提醒ID |
| data.customerId | Long | 客户ID |
| data.customerName | String | 客户名称 |
| data.reminderTime | DateTime | 提醒时间 |
| data.reminderPerson | String | 提醒人 |
| data.reminderContent | String | 提醒内容 |
| data.reminderType | Integer | 提醒方式 |
| data.reminderTypeDesc | String | 提醒方式描述 |
| data.reminderStatus | Integer | 提醒状态：0-未提醒 1-已提醒 2-已取消 |
| data.createTime | DateTime | 创建时间 |

**响应示例**：
```json
{
  "code": 200,
  "message": "新增成功",
  "data": {
    "id": 3001,
    "customerId": 1001,
    "customerName": "某某科技有限公司",
    "reminderTime": "2024-01-20 09:00:00",
    "reminderPerson": "张三",
    "reminderContent": "请跟进某某科技客户，准备上门演示",
    "reminderType": 3,
    "reminderTypeDesc": "系统内+短信",
    "reminderStatus": 0,
```
